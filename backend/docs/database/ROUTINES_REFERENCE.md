# Database Routines: Procedures, Functions & Events

This document details the programmable logic stored directly within the database server, including the stored procedures and automated events that drive the Graylinx analytics engine.

## 1. Analytics ETL Procedures
These procedures are dynamically generated by the backend based on the site configuration. They handle the "Transform" and "Load" stages of the ETL pipeline.

| Procedure Name | Scope | Description |
| :--- | :--- | :--- |
| **`proc_agg_CH{N}`** | Device (Chiller) | Pivots raw EAV data for Chiller N and calculates 15-min TR, kW, and SPC. |
| **`proc_agg_PV{N}`** | Device (Pump) | Calculates 15-min power and flow metrics for Primary Pumps. |
| **`proc_agg_CT{N}_F{M}`** | Device (Fan) | Aggregates Cooling Tower fan metrics. |
| **`proc_agg_plant`** | Site (Plant) | Sums up all device metrics into global Plant KPIs (Total Plant kW, Avg SPC). |
| **`proc_rollup_daily`** | Site (Daily) | Aggregates 15-minute data into daily totals for long-term reporting. |
| **`proc_run_all`** | Orchestrator | The "Master" procedure that calls all device, plant, and rollup procedures in sequence. |

## 2. Utility Procedures
| Procedure Name | Description |
| :--- | :--- |
| **`executeMyQueries`** | A generic utility procedure used by the backend to execute multiple SQL statements within a single connection. |

## 3. Database Events (Automation)
MySQL Events are used to trigger the ETL pipeline automatically without requiring an external task runner (like Cron).

| Event Name | Frequency | Action |
| :--- | :--- | :--- |
| **`ev_analytics_run`** | Every 15 Minutes | Calls `proc_run_all()` to refresh all analytics tables with the latest raw data. |

## 4. Implementation Details
The source code for these routines is generated by the **[generateAggregationProcedures.js](file:///d:/Harshan/graylinx-be/combined_repo/backend/Services/Gl_reports1/generateAggregationProcedures.js)** service. 

> [!NOTE]
> In the current multi-tenant setup, these routines are primarily active in production-ready databases like `pothys`. When a new site is commissioned, the backend automatically "provisions" these routines into the site's dedicated database.

## 5. Maintenance Commands
To manually trigger the ETL pipeline for a site, run the following SQL:
```sql
CALL proc_run_all();
```

To check the status of the automated event:
```sql
SHOW EVENTS LIKE 'ev_analytics_run';
```
